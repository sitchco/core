    **Create editor.jsx** at `modules/VideoBlock/blocks/video/editor.jsx` -- minimal skeleton:
    - Import `registerBlockType` from `@wordpress/blocks`
    - Import `useBlockProps`, `InnerBlocks`, `InspectorControls` from `@wordpress/block-editor`
    - Import `Placeholder` from `@wordpress/components`
    - Import `__` from `@wordpress/i18n`
    - Import `metadata` from `./block.json`
    - `Edit` function: uses `useBlockProps()`, renders a Placeholder with video icon prompting "Enter a video URL in the block settings" when no URL is set. When URL is set, renders a simple div (full preview comes in Plan 02). Always renders `<InnerBlocks />` (except modal-only mode which is also Plan 02).
    - `Save` function: returns `<InnerBlocks.Content />` wrapped in a div with `useBlockProps.save()`. CRITICAL: This must return exactly this -- any deviation breaks InnerBlocks persistence.
    - Call `registerBlockType(metadata.name, { edit: Edit, save: Save })`
    - Use `import React from 'react'` at the top for classic JSX transform compatibility.

    **Create render.php** at `modules/VideoBlock/blocks/video/render.php`:
    - Receives `$attributes`, `$content`, `$block` (standard WordPress render template variables)
    - If `$attributes['url']` is empty: echo `$content` directly (InnerBlocks only, no video wrapper -- NOOP-01)
    - If `$attributes['url']` is set: echo a wrapper `<div>` with `class="sitchco-video"` and data attributes (`data-url`, `data-provider`, `data-display-mode`, etc.) containing `$content`. This wrapper is the hook point for Phase 2 frontend JS.
    - Use `get_block_wrapper_attributes()` for the outer wrapper to include WordPress block supports (alignment classes).

    **Create style.css** at `modules/VideoBlock/blocks/video/style.css`:
    - Minimal styles: `.sitchco-video { position: relative; }` -- foundation for Phase 2 poster/overlay styling.

      - `test_block_type_is_registered`: Assert `WP_Block_Type_Registry::get_instance()->is_registered('sitchco/video')` is true
    - Test group: `// --- Render Output ---`
      - `test_render_without_url_outputs_innerblocks_content`: Create a minimal block render call with empty url attribute and sample $content. Assert output equals the $content string (no wrapper div).
      - `test_render_with_url_outputs_wrapper_with_data_attributes`: Create render call with url='https://youtube.com/watch?v=test' and sample $content. Assert output contains `sitchco-video` class, `data-url`, `data-provider="youtube"`, and the $content.
