  <action>
    Extend the `Edit` component in `editor.jsx` with the following:

    **1. Provider detection function** (module-level, outside component):
    ```jsx
    function detectProvider(url) {
        if (!url) return '';
        if (/(?:youtube\.com|youtu\.be)\//i.test(url)) return 'youtube';
        if (/vimeo\.com\//i.test(url)) return 'vimeo';
        return '';
    }
    ```

    **2. Slugify function** for modal ID generation:
    ```jsx
    function slugify(text) {
        return text.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '');
    }
    ```

    **3. URL input in InspectorControls:**
    - Add a `PanelBody` titled "Video Settings" as the first panel
    - Add a `TextControl` for the URL field with label "Video URL", placeholder "https://youtube.com/watch?v=..."
    - On URL change: set `url` attribute, auto-detect provider and set `provider` attribute
    - Do NOT trigger oEmbed fetch on every keystroke -- use debouncing (see below)

    **4. oEmbed fetch with debouncing:**
    - Use local component state: `oembedData` (object or null), `isLoading` (boolean), `error` (string or null)
    - Use a `useEffect` that watches the `url` attribute
    - Inside the effect, use a 500ms timeout (simple debounce pattern with `setTimeout`/`clearTimeout`) before fetching
    - Only fetch when URL is non-empty and looks valid (has a recognized provider)
    - Fetch via `apiFetch({ path: addQueryArgs('/oembed/1.0/proxy', { url }) })`
    - On success: store response in `oembedData` state, clear error
    - On error: set error message, clear oembedData
    - Include cleanup function to cancel the timeout on unmount or URL change

    **5. Auto-population of videoTitle and modalId** (AUTH-05):
    - When oEmbed response arrives successfully AND `_videoTitleEdited` is false: set `videoTitle` to `oembedData.title`
    - When oEmbed response arrives AND `_modalIdEdited` is false: set `modalId` to `slugify(oembedData.title)`
    - These auto-populate ONLY when the user hasn't manually edited them (tracked by `_videoTitleEdited` / `_modalIdEdited` boolean attributes)

    **6. Thumbnail preview with play icon overlay:**
    - When `oembedData` has `thumbnail_url`: render an `<img>` with the thumbnail inside the block preview area
    - Overlay a play icon SVG on top of the thumbnail, centered (50%/50% default, customizable position comes in Plan 03)
    - The play icon for now uses a simple generic dark play triangle SVG inline. Full provider-specific icons come in Plan 03.
    - When `isLoading`: show a `Spinner` component
    - When `error`: show error message text
    - When no URL: show the `Placeholder` with video icon (from Plan 01)
    - The thumbnail should NOT be an iframe or embedded player -- only a static image

    **7. Play icon SVG** (inline in the component for now):
    - Create a simple `PlayIcon` component that renders an SVG play triangle
    - Style it with absolute positioning over the thumbnail
    - Default size ~68px, semi-transparent dark background circle with white triangle
    - This is a temporary generic icon -- Plan 03 adds provider-specific branded icons

    **8. InnerBlocks rendering:**
    - `<InnerBlocks />` renders below the preview area
    - No `allowedBlocks` restriction (any block type allowed)

    **CSS in style.css** -- add editor preview styles:
    - `.sitchco-video__preview`: relative positioning, overflow hidden
    - `.sitchco-video__thumbnail`: width 100%, display block, object-fit cover
    - `.sitchco-video__play-icon`: absolute positioned, centered (transform translate -50%/-50%), pointer-events none in editor
    - `.sitchco-video__placeholder`: styling for the no-URL state
    - `.sitchco-video__error`: error message styling (red text)
    - `.sitchco-video__loading`: loading state with centered spinner

    **Anti-patterns to avoid:**
    - Do NOT use `@wordpress/compose`'s `useDebounce` -- it debounces the value, not the callback, which can cause stale closures. Use a simple setTimeout/clearTimeout pattern in useEffect instead.
    - Do NOT import React hooks from `react` -- import `useState`, `useEffect`, `useCallback`, `useRef` from `@wordpress/element` (WordPress convention).
    - Do NOT embed an iframe for preview -- only the oEmbed thumbnail image.
    - Do NOT fire oEmbed requests for unrecognized providers (only YouTube and Vimeo).
