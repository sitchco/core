<task type="auto">
  <name>Task 1: Play icon configuration controls and branded SVG icons</name>
  <files>modules/VideoBlock/blocks/video/editor.jsx, modules/VideoBlock/blocks/video/style.css</files>
  <action>
    Extend the `Edit` component in `editor.jsx` with play icon configuration:

    **1. YouTube branded play icon SVGs** (AUTH-09):
    YouTube API ToS requires branded play buttons. Create inline SVG components or SVG markup strings for:
    - **YouTube Dark**: Red play button on semi-transparent dark background (the standard YouTube play overlay). SVG: rounded rectangle with YouTube red (#FF0000) background, white triangle play symbol. Dimensions approximately 68x48px.
    - **YouTube Light**: Same shape but with semi-transparent white/light background instead of dark.
    - **YouTube Red**: Solid red (#FF0000) background with white triangle.

    These are simplified representations of the YouTube branded play button. The SVGs should be recognizable as YouTube-style but are SVG recreations (not raster copies).

    **Generic play icon SVGs:**
    - **Generic Dark**: Circle with semi-transparent dark (#000000cc) background, white triangle
    - **Generic Light**: Circle with semi-transparent white (#ffffffcc) background, dark triangle

    Create a `getPlayIconSvg(provider, style)` function that returns the appropriate SVG JSX based on provider and selected style.

    **2. Play icon style selector** (AUTH-09):
    - Add a new `PanelBody` titled "Play Icon" in the inspector
    - Only show this panel when a URL is set (no play icon config needed without a video)
    - Add a `SelectControl` for "Icon Style"
    - Options are **provider-conditional**:
      - When `provider === 'youtube'`: options are `dark`, `light`, `red` (all YouTube-branded)
      - When `provider !== 'youtube'` (including Vimeo and empty): options are `dark`, `light` (generic)
    - Stores to `playIconStyle` attribute
    - When provider changes (e.g., user switches from YouTube URL to Vimeo URL), if the current `playIconStyle` is `'red'` (YouTube-only), auto-reset to `'dark'`

    **3. Play icon position sliders** (AUTH-10):
    - Add two `RangeControl` components in the "Play Icon" panel:
      - "Horizontal Position" (X): min 0, max 100, step 1, default 50. Stores to `playIconX`.
      - "Vertical Position" (Y): min 0, max 100, step 1, default 50. Stores to `playIconY`.
    - Help text: "Position as percentage (50% = centered)"

    **4. Click behavior toggle** (AUTH-11):
    - Add a `SelectControl` in the "Play Icon" panel:
      - Label: "Click Behavior"
      - Options: `[{ label: 'Entire poster', value: 'poster' }, { label: 'Play icon only', value: 'icon' }]`
      - Stores to `clickBehavior` attribute
      - Help text: "Controls whether clicking anywhere on the poster or only the play icon starts the video"

    **5. Update the play icon in the editor preview:**
    - Replace the temporary generic PlayIcon from Plan 02 with the `getPlayIconSvg()` function
    - The play icon position in the preview should use inline styles based on `playIconX` and `playIconY`:
      ```jsx
      style={{
          position: 'absolute',
          left: `${playIconX}%`,
          top: `${playIconY}%`,
          transform: 'translate(-50%, -50%)',
      }}
      ```
    - As the author drags the sliders, the play icon should visually move on the thumbnail preview in real time
    - The icon variant should update immediately when the style selector changes

    **6. CSS updates in style.css:**
    - `.sitchco-video__play-icon--youtube-dark`, `--youtube-light`, `--youtube-red`: any variant-specific sizing or styling
    - `.sitchco-video__play-icon--generic-dark`, `--generic-light`: generic icon variant styles
    - Ensure the play icon is properly sized and visible on various thumbnail aspect ratios

    **Anti-patterns to avoid:**
    - Do NOT use raster images (PNG/JPG) for play icons -- use inline SVGs for crisp rendering at any size
    - Do NOT allow the "red" style option when provider is not YouTube -- this is a legal requirement
    - Do NOT make the play icon clickable in the editor -- it's a visual preview only. `pointer-events: none` on the icon.
