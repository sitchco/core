<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Browser verification of modal playback</name>
  <files></files>
  <action>
Present the verification checklist to the user for browser testing of complete modal integration.

Setup: Create or edit a test page in WordPress with the following video blocks:

1. Modal-mode YouTube block -- Add a sitchco/video block, paste a YouTube URL, set display mode to "Modal"
2. Modal-mode Vimeo block -- Add a sitchco/video block, paste a Vimeo URL, set display mode to "Modal"
3. Modal-only block -- Add a sitchco/video block, paste a YouTube URL, set display mode to "Modal Only", note the modal ID
4. Trigger link -- Add a paragraph or button block with a link pointing to #[modal-id] matching the modal-only block's ID
5. Inline block -- Add a sitchco/video block with display mode "Inline" (regression check)

Test steps:

1. Modal open (MODL-01): Click the play button on the modal-mode YouTube poster. Verify a dialog opens with the video playing inside.
2. Hash update (MODL-02): Check the browser URL bar -- it should show #[modal-id] in the URL.
3. Poster stays (MODL-04): Look behind the dialog backdrop -- the poster on the page should still be visible (not hidden or replaced).
4. Close pauses (MODL-05): Click the X button to close the modal. Verify audio stops immediately. Open browser DevTools > Elements and confirm the iframe is still in the dialog DOM (not removed).
5. Resume on reopen (MODL-06): Click the same poster again. Verify the modal reopens and video resumes from where it was (no new loading state, no duplicate iframes).
6. Escape key pause: Open a modal, then press Escape. Verify audio stops immediately (this specifically tests the native close event handling).
7. Backdrop click pause: Open a modal, click the dark backdrop area outside the video. Verify modal closes and audio stops.
8. Vimeo modal: Repeat steps 1, 4, 5, 6 with the Vimeo modal-mode block.
9. Modal-only invisible (MODL-07): Verify the modal-only block has no visible element on the page -- the page should show nothing where the block is.
10. Trigger link (TRIG-01): Click the trigger link pointing to #[modal-id]. Verify the video modal opens and video plays.
11. Deep link (TRIG-03): Copy the page URL and add #[modal-id] hash. Open in a new tab. Verify the modal opens automatically and video starts playing.
12. ARIA check (ACCS-04): In DevTools, inspect the dialog element. Verify it has aria-labelledby pointing to an h2 element containing the video title.
13. Inline regression: Click the inline-mode video poster. Verify it plays inline as before (no modal, poster replaced by player).
  </action>
