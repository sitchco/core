Add the following module-level state alongside the existing `modalPlayers` Map:

1. **activePlayers Map** — `new Map()` keyed by provider video ID string. Entry shape: `{ player: SDKInstance, provider: 'youtube'|'vimeo', url: string }`.

2. **pausePlayerById(videoId)** function — looks up entry in activePlayers by videoId, calls `entry.player.pauseVideo()` for youtube or `entry.player.pause()` for vimeo. No-ops if videoId not found.

3. **registerActivePlayer(videoId, player, provider, url)** function — iterates activePlayers and pauses every entry where `id !== videoId` (mutual exclusion per MXCL-01). Then sets `activePlayers.set(videoId, { player, provider, url })`.

4. **video-request-pause subscriber** — at the bottom of the file alongside existing hook registrations, add:
   `sitchco.hooks.addAction('video-request-pause', function(videoId) { pausePlayerById(videoId); stopMilestonePolling(videoId); }, 10, 'video-block');`
   (EXTN-02, NOOP-02 — external code uses this instead of auto-pause on visibility)

5. **Extend createYouTubePlayer** — add `onStateChange` to the `events` object in the `new YT.Player()` config:
   - `YT.PlayerState.PLAYING`: call `registerActivePlayer(videoId, event.target, 'youtube', url)`, fire `sitchco.hooks.doAction('video-play', { id: videoId, provider: 'youtube', url: url })`, call `startMilestonePolling(videoId, event.target, 'youtube', url)`.
   - `YT.PlayerState.PAUSED`: fire `sitchco.hooks.doAction('video-pause', { id: videoId, provider: 'youtube', url: url })`, call `stopMilestonePolling(videoId)`.
   - `YT.PlayerState.ENDED`: fire `video-progress` with milestone 100, fire `video-ended`, call `stopMilestonePolling(videoId)`, call `activePlayers.delete(videoId)`.
   Note: The `url` variable must be available in the closure. Add a `url` parameter to `createYouTubePlayer` (5th param) OR derive it by having call sites pass it. The simpler approach: add `url` and `displayMode` as additional parameters. Call sites already have both values available.

6. **Extend createVimeoPlayer** — after `player.ready().then(...)`, add event bindings:
   - `player.on('play', ...)`: call `registerActivePlayer(videoId, player, 'vimeo', url)`, fire `video-play`, call `startMilestonePolling(...)`.
   - `player.on('pause', ...)`: fire `video-pause`, call `stopMilestonePolling(videoId)`.
   - `player.on('ended', ...)`: fire `video-progress` milestone 100, fire `video-ended`, call `stopMilestonePolling(videoId)`, call `activePlayers.delete(videoId)`.
   Same parameter additions as YouTube.

7. **Update call sites** — `handlePlay()` and `handleModalShow()` both call `createYouTubePlayer`/`createVimeoPlayer`. Pass `url` and `displayMode` from the dataset attributes already read at those call sites:

   - In `handleModalShow`: `createYouTubePlayer(playerContainer, videoId, startTime, modalId, url, 'modal')` and same for Vimeo.

When `pausePlayerById` calls `pauseVideo()` or `pause()`, the SDK fires its native pause event, which triggers the `onStateChange(PAUSED)` / Vimeo `pause` listener, which fires `video-pause`. So `video-pause` fires naturally.

**CRITICAL — Pitfall 5:** `displayMode` is not currently a parameter of `createYouTubePlayer`/`createVimeoPlayer`. Add it as a parameter so it's available for the JS filter context object. The call sites already have it.

**Milestone Polling (ANLT-02):**

1. Add module-level state:
   - `const pollIntervals = new Map()` — videoId -> intervalId
   - `const milestonesFired = new Map()` — videoId -> Set of fired percentages (25, 50, 75)
   - `const MILESTONES = [25, 50, 75]` — 100 is handled by the ended event, not polling

2. **startMilestonePolling(videoId, player, provider, url)** function:
   - If `pollIntervals.has(videoId)`, return early (already polling).
   - If `!milestonesFired.has(videoId)`, initialize: `milestonesFired.set(videoId, new Set())`.
   - Create `setInterval` at 1000ms:
     - YouTube branch (synchronous): `const current = player.getCurrentTime(); const duration = player.getDuration(); checkMilestones(videoId, provider, url, current, duration);`
     - Vimeo branch (async): `Promise.all([player.getCurrentTime(), player.getDuration()]).then(function([current, duration]) { checkMilestones(videoId, provider, url, current, duration); }).catch(function() {});` — catch silently since player may be destroyed mid-poll.
   - Store interval ID: `pollIntervals.set(videoId, intervalId)`.

3. **stopMilestonePolling(videoId)** function:
   - Get intervalId from pollIntervals, `clearInterval` it, `pollIntervals.delete(videoId)`.
   - Do NOT clear milestonesFired — milestones fire at most once per page load per CONTEXT.md decision.

4. **checkMilestones(videoId, provider, url, current, duration)** function:
   - Guard: if `!duration || duration <= 0` return.
   - Compute `pct = (current / duration) * 100`.
   - Get `fired = milestonesFired.get(videoId)`.
   - For each milestone in MILESTONES: if `pct >= milestone && !fired.has(milestone)`, add to fired set and call `sitchco.hooks.doAction('video-progress', { id: videoId, provider: provider, url: url, milestone: milestone })`.
   - This handles seeking past milestones correctly (all crossed milestones fire).

**JS Filters (EXTN-04, EXTN-05):**

5. In `createYouTubePlayer`, before `new YT.Player(target, ...)`:
   - Build `defaultPlayerVars` object (the existing playerVars object literal).
   - Apply filter: `var playerVars = sitchco.hooks.applyFilters('sitchco/video/playerVars/youtube', defaultPlayerVars, { url: url, videoId: videoId, displayMode: displayMode });`
   - Use filtered `playerVars` in the `new YT.Player()` config.

6. In `createVimeoPlayer`, before `new Vimeo.Player(target, ...)`:
   - Build `defaultOptions` object (the existing options object literal).
   - Apply filter: `var options = sitchco.hooks.applyFilters('sitchco/video/playerVars/vimeo', defaultOptions, { url: url, videoId: videoId, displayMode: displayMode });`
   - Use filtered `options` in `new Vimeo.Player(target, options)`.
